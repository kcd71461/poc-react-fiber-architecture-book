# Chapter Review Skill

## 목적

React Fiber Architecture 전자책의 챕터를 **한국어 독자(React 중급자 기준)** 관점에서 리뷰하고,
개선이 필요한 부분을 구조화된 마크다운 피드백 문서로 작성한다.

---

## 실행 절차

### Step 1. 페이지 콘텐츠 수집

`browser_subagent`를 사용해 주어진 URL의 페이지 전체 콘텐츠를 읽는다.

- 페이지를 끝까지 스크롤하여 모든 요소(제목, 본문, 코드 블록, 테이블, 체크리스트 등)를 수집한다.
- 인터랙티브 요소(시뮬레이션, 시각화 컴포넌트)가 있으면 존재 여부와 설명 텍스트도 기록한다.

### Step 2. 독자 관점 리뷰 기준 적용

아래 **7가지 리뷰 관점**을 기준으로 분석한다:

#### 2-1. 전체 구조 및 흐름

- 섹션 순서가 논리적으로 연결되는가?
- 서론 → 문제 정의 → 해결책 → 실습 → 요약의 학습 흐름이 지켜지는가?
- 섹션 간 전환이 자연스러운가? (연결 문장이 있는가?)

#### 2-2. 문장 및 표현 오류

- 주어 누락, 문장 중복, 어색한 연결 등 **편집 오류**가 있는가?
- 한국어 사용자가 이해하기 어려운 번역투 표현이 있는가?

#### 2-3. 용어 및 개념 설명 적절성

- 핵심 용어(예: Stack Reconciler, Fiber, Reconciler 등)가 처음 등장할 때 정의되는가?
- 정의 위치가 해당 용어의 첫 등장 시점과 맞는가?
- 심화 개념(예: Algebraic Effects)은 적절한 난이도 표시 또는 쉬운 비유가 함께 제공되는가?

#### 2-4. 코드 예제 가독성

- 코드 블록에 인라인 주석이 있는가?
- 코드가 실행되기 전에 코드의 전제 조건(예: 자료구조 구성)이 설명되는가?
- 의사 코드(pseudo-code)와 실제 구현 코드가 명확히 구분되는가?

#### 2-5. 시각 자료 및 인터랙티브 요소 연결

- 시뮬레이션, 다이어그램 등 UI 요소가 존재할 경우, 해당 요소를 안내하는 텍스트가 있는가?
- 시각 자료 없이 텍스트만으로 설명이 충분한가?

#### 2-6. 섹션 위치의 적절성

- "다음 장 예고"는 본문 끝(요약 뒤)에 위치하는가?
- "역사적 맥락", "배경 정보"는 서론에 가깝게 위치하는가?
- "오해 바로잡기"는 본문 내용을 충분히 소화한 후에 등장하는가?

#### 2-7. 독자 학습 지원 요소

- 요약 체크리스트가 Learning Objective와 일치하는가?
- 실습 예제가 본문 개념과 연결되는가?
- Source Map 등 심화 탐색 경로가 제공되는가?

### Step 3. 피드백 문서 작성

분석 결과를 바탕으로 아래 **문서 템플릿**에 따라 마크다운 파일을 작성한다.

#### 출력 파일 경로 규칙

```
reviews/chapter-{번호}-review.md
```

예시:

- `reviews/chapter-01-review.md`
- `reviews/chapter-02-review.md`

#### 문서 템플릿

```markdown
# 📖 {챕터 제목} — 독자 관점 리뷰

- **리뷰 대상**: `{URL 경로}`
- **리뷰 일자**: {YYYY-MM-DD}
- **리뷰어 관점**: 독자 (React Fiber Architecture 학습 중인 한국어 사용자, 중급 수준)

---

## ✅ 잘 된 점 (강점)

{강점 항목을 bullet로 나열. 최소 3개 이상.}

---

## ⚠️ 수정이 필요한 부분

{각 항목은 아래 형식으로:}

> 현재 텍스트 또는 현재 구조 설명

{구체적인 문제 설명}

**제안:**
{개선 방향 또는 수정 예시}

---

## 🗂️ 수정 우선순위 요약

| 우선순위 | 항목 | 유형                    |
| :------: | ---- | ----------------------- |
| 🔴 높음  | ...  | 편집 오류 / 구조 오류   |
| 🟠 중간  | ...  | 내용 보완 / 구조 개선   |
| 🟡 낮음  | ...  | 독자 배려 / 가독성 개선 |

{전반적인 평가와 핵심 권고 사항을 3~5문장으로 요약.}
```

## 주의 사항

1. **독자 관점 유지**: 저자/개발자 관점이 아닌, 이 책으로 React Fiber를 **처음 학습하는 한국어 독자** 관점에서 판단한다.
2. **근거 기반 피드백**: 모든 피드백은 원문 텍스트를 직접 인용하거나 구체적인 섹션을 지정해 근거를 명시한다.
3. **건설적 제안 필수**: 문제 지적만 하지 않고, 반드시 개선 방향이나 수정 예시를 함께 제시한다.
4. **한국어로 작성**: 피드백 문서 전체를 한국어로 작성한다. 기술 용어(Fiber, Reconciler 등)는 원문 그대로 사용한다.
5. **우선순위 분류**:
   - 🔴 높음: 오탈자, 주어 누락, 문장 중복 등 즉시 수정이 필요한 오류
   - 🟠 중간: 구조 개선, 설명 보강 등 콘텐츠 품질에 영향을 주는 항목
   - 🟡 낮음: 독자 경험 향상을 위한 선택적 개선 항목
6. **점진적 집필 환경 인식**: 이 전자책은 챕터를 순차적으로 작성 중이다. **현재 마지막 챕터의 하단 네비게이션 "다음" 링크가 용어 사전(`/book/glossary/`)으로 연결되는 것은 정상 동작**이다. 아직 작성되지 않은 다음 챕터가 없을 경우 용어 사전이 대신 연결된다. 이를 오류나 수정 대상으로 지적하지 않는다. 단, 이미 해당 챕터가 존재하는데 잘못 연결된 경우(예: 03장이 있는데 01장에서 다음이 용어 사전으로 연결)는 수정 대상으로 지적한다.
